# Цепочка акторов

Цепочка акторов - упорядоченная история участников действия в агентной системе: исходный пользователь, промежуточные агенты, gateway и downstream-инструменты.

## Смысл

В multi-agent workflow действие редко выполняется одним субъектом. Пользователь ставит задачу, один агент планирует, другой агент исследует, третий вызывает инструмент или создает pull request. Если каждый hop видит только непосредственного caller, система теряет provenance.

Цепочка акторов делает делегирование переносимым: downstream-система может оценивать не только последнего caller, но и всю линию "кто попросил, кто действовал, через кого прошел запрос".

## Исследовательский интерес

Для [[Наблюдаемость|наблюдаемости]] actor chain является trace не только технических вызовов, но и ответственности. Это расширяет управляемость: действие можно связать с инициатором, агентом, инструментом, authorization decision и контекстом инцидента.

Для [[Протоколы архитектуры|архитектурных протоколов]] actor chain интересна как часть runtime-контракта. Протокол должен не только доставить данные, но и сохранить контекст делегирования через границы компонентов.

## Риски и лимиты

- Рост размера токенов и логов.
- Latency overhead на per-hop token exchange.
- Сложность policy language, если политики учитывают пользователя, агента, intent, tool и риск.
- Privacy-риск: actor chain может переносить чувствительный контекст слишком далеко.
- Хрупкость при частичной миграции: один legacy hop может оборвать provenance.
