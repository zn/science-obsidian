# Протоколы архитектуры

Протоколы архитектуры - общие правила, форматы, интерфейсы и процедуры, через которые компоненты сложной системы становятся совместимыми.

## Смысл

Протокол ограничивает локальное поведение компонента, но расширяет глобальные возможности системы. Компоненты могут быть разными, если они соблюдают общий способ соединения, обмена или сборки.

Примеры в программной инженерии:

- API contract;
- сетевой протокол;
- формат сообщения;
- схема данных;
- соглашение о backward compatibility;
- deployment protocol;
- migration protocol.

## Хрупкость протоколов

Протокол часто является fixed point архитектуры. Это делает его ценным, но опасным:

- изменение протокола дорого;
- нарушение протокола имеет большой blast radius;
- атака на общий протокол может обойти защиту отдельных компонентов;
- слишком слабый протокол плохо удерживает совместимость;
- слишком сильный протокол может подавлять evolvability.

В агентных системах к протоколам добавляется перенос контекста делегирования: [[Цепочка акторов|цепочка акторов]], scoped token, audience, срок действия полномочия и проверка [[Идентичность AI-агента|идентичности агента]] становятся частью runtime-контракта между компонентами.

## Исследовательский интерес

Для кандидатской темы протоколы могут стать операциональным языком лимитов: через них можно изучать, какие ограничения делают систему управляемой, а какие превращаются в источник хрупкости.
